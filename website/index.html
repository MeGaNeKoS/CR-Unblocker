Announcements

New Ownership & Version 4.0.0 Release

Hello everyone!

As of version 4.0.0, CR-Unblocker is maintained by めがねこ. The backend and proxy server have been fully migrated and are now operated at us.community-proxy.meganeko.dev.

Status & Monitoring

You can check live service availability on our official monitor at community-proxy.meganeko.dev/monitor. This page also powers the alerts sent to our uptime robot. For those who need the raw HAProxy statistics, head to community-proxy.meganeko.dev/stats.

Notice: The previous maintainer, Finn, has retired and the old unblocker server is no longer available. I want to extend a huge thank you to Finn for his dedication and years of work on this project, as well as to everyone who supported CR-Unblocker along the way!

About version 3.2.0: The 3.2.0 release was accidentally deleted from AMO during transition. Sorry for the inconvenience. If you need a copy of this version for historical reasons, you can manually rebuild it from commit 73ec3576378bf4ddba617a6e0e0ad9d2d540b6e6 on our GitHub repository.

To check out this version locally:
git clone https://github.com/MeGaNeKoS/CR-Unblocker.git && cd CR-Unblocker && git checkout 73ec3576378bf4ddba617a6e0e0ad9d2d540b6e6

Changelog

4.0.0 – Major update and new proxy server

  • Switched default proxy server to us.community-proxy.meganeko.dev
  • Now proxies only geo-restricted Crunchyroll traffic, direct for all other traffic
  • No more session ID manipulation—unblocks anime directly
  • Improved support for all regions, including Japan

3.2.0 – New default proxy and improved access for restricted regions

This release changed the default proxy and improved proxy handling.

  • Changed default proxy endpoint
  • Reduced load on proxy availability check
  • Proxied initial website load to allow access from restricted regions like Japan

3.1.0 – HTTP proxy support and status check

Added support for HTTP proxies and UI improvements with proxy availability checks.

3.0.2 – Error handling

  • Added notification for incorrect proxy settings
  • Fallback if proxy times out

3.0.1 – Housekeeping update

  • Removed legacy code
  • Hide password for custom proxy config
  • Fixed checkbox setting for custom proxy
  • Use DNS of proxy service

3.0.0 – Crunchyroll Beta support

  • Proxy requests for new token endpoint with SOCKS proxy
  • Custom proxy configuration options
  • Removed legacy login logic
  • Modernized for Crunchyroll Classic

2.3.1

Most issues seem to be errors getting new sessions. I think this is largely related to the reload loop. To improve on this I implemented a timer to only reload once a minute - if the session can not be obtained, it is unlikely to work a few seconds after.

  • Added timer between session reloads

2.3
